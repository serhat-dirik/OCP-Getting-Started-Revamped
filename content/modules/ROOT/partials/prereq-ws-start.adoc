[NOTE]
.Before you start — your environment is already set up
====
This module is self-contained: its world — namespaces, applications, Git repositories, pipelines, and access — is materialized for you automatically by the workshop setup, so you do *not* need to have completed any earlier module.

Confirm it is ready from the *Terminal* tab. The command prints one row naming the `ws-entry-{module-id}` marker (its `DATA` column reads `2`):

[source,sh,role=execute,subs="attributes"]
----
oc get cm ws-entry-{module-id} -n {user}-dev
----

If it instead returns `Error ... not found`, the environment has not been materialized yet — tell your instructor, who runs `ws start {module-id} --user {user}`.
====
