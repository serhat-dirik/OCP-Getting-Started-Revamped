[NOTE]
.Before you start — materialize this module's environment
====
This module is self-contained.
Its world — namespaces, applications, Git repositories, pipelines, and access — is created for you on demand, so you do *not* need to have completed any earlier module.

Open the *Terminal* tab and run:

[source,sh,role=execute,subs="attributes"]
----
ws start {module-id}
----

*What you should see:* a line reading `✅ {module-id} ready for {user}` once every component reports healthy (usually under a minute).
If it stalls, run `ws verify {module-id}` for a component-by-component check, or `ws reset {module-id}` to wipe and start clean.

The command is safe to re-run: it re-materializes the same state without duplicating anything.
====
